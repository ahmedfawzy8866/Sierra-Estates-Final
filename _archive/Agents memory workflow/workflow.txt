# Sierra Blu Realty 2027: Agents and Workflows Overview
This document provides a comprehensive overview of the Intelligence Layer (Agents) and the External Automation Scripts (Workflows) that power the Sierra Blu 2027 platform.
---
## 🤖 The Intelligence Layer (Four-Agent System)
The system utilizes an AI-driven, four-agent architecture to handle specialized tasks, ensuring code quality, data accuracy, and automated business operations.
### 1. The Scribe 📝 (Documentation)
**Role:** Technical Documentation & API Maintenance
*   Maintains API specifications and system documentation.
*   Documents breaking changes across the codebase.
*   Generates integration guides for new developers or external stakeholders.
### 2. The Curator 🎨 (Design QA)
**Role:** Design System Enforcement & Visual Quality Assurance
*   Enforces Tailwind color consistency and adherence to the design tokens.
*   Reviews component hierarchies and spatial rhythms (glassmorphism, cinematic animations).
*   Validates typography standards to ensure a premium, weightless UI aesthetic.
### 3. The Matchmaker 🤝 (Intelligence & Lead Matching)
**Role:** Investor-Property Matching & Recommendation Engine
*   Scores available properties against investor profiles.
*   Answers "Leila's 3 Gold Questions":
    1.  **Intent:** What is the investment goal?
    2.  **Capital:** What is the budget and financing structure?
    3.  **Timeline:** How fast do they want to close?
*   Generates comprehensive AI match reports to assist brokers.
### 4. The Closer 💼 (Deal Automation)
**Role:** Transaction & Proposal Orchestration (Stage 9)
*   Manages proposal generation and creates PDF documents.
*   Tracks contract states from draft to closed/cancelled.
*   Initiates e-signature workflows (e.g., via DocuSign).
*   Generates revenue and analytics reports.

## ⚙️ External Workflows (n8n & Node Scripts)
The platform houses five automated data ingestion and communication workflows under the `workflows/` directory. They sync data between Google Sheets, WhatsApp, Property Finder, and Firebase.
### 01. WhatsApp Scraper (`workflows/01-whatsapp-scraper`)
*   **Purpose:** Monitors designated Egyptian broker WhatsApp groups (e.g., `مجموعة وسطاء التجمع`) for property listings.
*   **Action:** Writes raw message flows (Timestamp, From, Role, Message, HasMedia) into the `raw_messages` tab in Google Sheets for parsing.
*   **Trigger:** Manual / Continuous polling (`npm run whatsapp-scraper`).
### 02. Owner Search (`workflows/02-owner-search`)
*   **Purpose:** Automates the search for direct-owner properties.
*   **Action:** Queries the Property Finder & OLX API for Tagamoa/New Cairo direct-owner listings and appends the candidates into the `owner_leads` Sheets tab.
*   **Schedule:** Daily at 9:00 AM (cron: `0 9 * * *`).
### 03. Owner Contact (`workflows/03-owner-contact`)
*   **Purpose:** Initial outreach to scraped property owners.
*   **Action:** Sends automated bilingual WhatsApp scripts to direct owners to confirm property availability and tracks delivery status.
*   **Schedule:** Daily at 10:00 AM (cron: `0 10 * * *`).
### 04. Email Sender (`workflows/04-email-sender`)
*   **Purpose:** Stakeholder communication and marketing.
*   **Action:** Sends bulk emails, investor briefings, matches, and closing document summaries via SendGrid based on data in the `email_campaigns` Sheets tab.
*   **Schedule:** Daily at 8:00 AM (cron: `0 8 * * *`).
### 05. Unit Adder (`workflows/05-unit-adder`)
*   **Purpose:** Database synchronization and data cleanup.
*   **Action:** Reads new properties from the `new_units` Sheets tab, runs fuzzy spelling correction on compound names, deduplicates the listings, and writes them to the Firestore `listings` database.
*   **Schedule:** Every 30 minutes (cron: `*/30 * * * *`).
